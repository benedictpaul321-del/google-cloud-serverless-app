Multi-format parsing

Handles PDF, plain text, and images. Uses pypdf for PDFs and simulated OCR for scanned images.

🏷️

Auto-tagging

Heuristic keyword extraction generates category tags — financial, medical, legal — based on content.

⚡

Fully serverless

Cloud Run scales to zero when idle. No VMs, no container orchestration, pay only for what you use.

🔁

Idempotent deploy

The deploy scripts check for existing resources before creating them — safe to re-run at any time.

🧪

Local dev mode

Run the full pipeline locally with LOCAL_DEV=true. No GCP credentials needed for testing.

🔐

GitHub Actions ready

Includes a setup script that provisions a least-privilege service account and outputs a deploy key.

BigQuery output schema

Field Type Mode Description
gcs_uri STRING REQUIRED Full GCS URI of the processed file
filename STRING REQUIRED Original file name
file_type STRING NULLABLE Detected format or extension
processed_at TIMESTAMP REQUIRED When the document was processed
word_count INTEGER NULLABLE Total words parsed from text
tags STRING REPEATED Heuristic category tags array
extracted_text STRING NULLABLE Full extracted or mock OCR text

Quickstart

01

Authenticate with GCP

Log in with the gcloud CLI and set your target project.

gcloud auth login gcloud config set project YOUR_PROJECT_ID
02

Run the deploy script

The script enables APIs, creates all GCP resources, builds the container, and wires up Pub/Sub — in one command.

# macOS / Linux chmod +x deploy.sh && ./deploy.sh # Windows PowerShell .\deploy.ps1
03

Upload a document

Drop any PDF, TXT, or image into the provisioned bucket to trigger the pipeline.

gcloud storage cp my_file.pdf gs://YOUR_PROJECT_ID-document-ingestion/
04

Query results in BigQuery

Processed metadata and extracted text appear in BigQuery within seconds.

SELECT * FROM `document_processing.metadata` ORDER BY processed_at DESC LIMIT 10;
